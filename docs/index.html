Project Landing

A container runtime that treats lifecycle evidence as first-class output.

GardenLiminal is an open Rust runtime for workload isolation with structured lifecycle events and a visible hardening roadmap. It is the runtime layer of the Liminal Stack.

Review stack demo View NLnet overlay

Why it exists

ProblemMost runtimes focus on execution first and evidence later, leaving the execution boundary hard to inspect.
ConsequenceAudit trails get scattered across tools and it becomes harder to explain what actually happened at runtime.
ContributionGardenLiminal makes lifecycle evidence part of runtime design rather than an afterthought.

Why it matters

Open runtime logic

Isolation and event logic are visible and reviewable in an open Rust codebase.

Execution plus evidence

The runtime is positioned not only as an execution engine, but as an evidence producer.

Stack fit

GardenLiminal bridges workload execution and persistent event history through LiminalDB integration.

What to remember

  • Trustworthy infrastructure needs a runtime layer that can show what happened at execution time.
  • Open isolation is stronger when it is paired with structured runtime evidence.
  • GardenLiminal exists to make that boundary visible, inspectable, and reusable.
